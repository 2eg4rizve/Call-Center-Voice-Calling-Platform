# Repository Instructions

## Local AI Documents

- The `ai_docs/` directory contains local project requirements, plans, and other context intended for AI-assisted work.
- Consult relevant files in `ai_docs/` before planning, analyzing, designing, or implementing project features.
- Treat documents in `ai_docs/` as project context even though Git ignores the directory.
- The contents of `ai_docs/` must remain local: never stage, commit, or push them.
- When a prompt refers to an AI document without its complete path, inspect `ai_docs/` to locate the relevant file.

## Git Commit Rules

- Never add `Co-authored-by` or any other co-author trailer to a commit.
- Never mention ChatGPT, Codex, OpenAI, AI assistance, or generated-by tooling in commit messages or commit trailers.
- Write concise, conventional commit messages that describe only the repository change.
